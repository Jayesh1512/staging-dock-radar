What Changed from Original Spec

Every decision below corresponds to a badge in the UI above. Refer to phase1-gap-analysis.md for full reasoning.

  • Removed LLM Model parameter removed from all config bars. Model choice is an infrastructure concern, not a per-run UX decision in Phase 1.
  • Changed Min Score moved from Step 2 → Step 1 config bar. Editable at collection time; read-only display in Step 2. Fixes the state drift problem (GAP 2) where re-filtering after pruning couldn't rehydrate dropped articles.
  • Removed Review Gate toggle removed from Step 1 config bar. Articles always auto-flow to Step 3 when scoring completes. The gate concept is replaced by the Reviewed Inbox sub-view in Step 3, which provides post-hoc visibility without blocking the pipeline.
  • Changed Step 2 config bar: all parameters now read-only. Configuration is locked at collection time. Step 2 only shows the run selector and locked display values.
  • New Sources panel in Step 1 — Google News active; LinkedIn and Facebook shown as "coming soon" disabled checkboxes. Makes the source architecture visible without implying current functionality (GAP 11).
  • New Queue redesigned as batch sections per run — articles are grouped by the collection run that discovered them. Each batch has its own label showing run name, date/time, and signal count.
  • Changed Global "Select All" removed from queue header. Per-batch Select All + Bulk Dismiss replaces it. Bulk dismiss shows a toast with "X articles dismissed" count (GAP 6).
  • New Batch dividers show run name, date, and signal count in a centered horizontal rule. Visually separates articles from different collection runs.
  • Changed Multi-action model: Slack and Bookmark no longer exit the queue. Article stays in the Active Queue after Slack or Bookmark. Only "Mark as Reviewed" moves an article to the Reviewed inbox. Dismiss permanently removes it. This eliminates the re-bookmarking edge case (GAP 18) and makes article state explicit.
  • Removed Sent/Bookmarked collapsible sections removed. Replaced by the Reviewed Inbox sub-view — a tab inside Step 3 (Active Queue | Reviewed). The Reviewed view shows a flat table filterable by All / Slacked / Bookmarked / Date, with action icons per article. This gives full post-action visibility without an archival sidebar.
  • New Article drawer person cards include empty Phase 2 slot — each person shows a dashed placeholder for future email / LinkedIn enrichment. No mock data filled in Phase 1 (GAP 12 — forward-compatible).
  • New Empty states defined for all steps — Queue: "All caught up — no new signals to review". Step 2 all-dismissed: table stays with 0 rows + "All articles dismissed". Failures surface as toast only, no retry button in Phase 1 (GAP 8).
  • New Scoring progress shows cached article count — "X already cached from previous runs" shown below progress bar. Transparent smart-memory behaviour (from GAP 5 run-selector design).
  • Removed "Proceed to Queue" button removed from Step 2 — no longer needed as the Review Gate toggle is removed. Articles auto-flow to the queue on scoring completion.
  • New SOURCE_BADGE_COLORS fully defined — Google News (yellow #FEF9C3 / #A16207), LinkedIn (blue #DBEAFE / #1E40AF), Facebook (indigo #EEF2FF / #4338CA). All three defined in constants, not just Google News (GAP 11).
  • Changed Dismiss is immediate and permanent — no undo, no confirmation dialog. Single dismiss: silent. Bulk dismiss: shows toast "X articles dismissed". Low-friction design for fast BD triage (GAP 6).
