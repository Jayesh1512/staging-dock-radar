5. Deep Dive: Government Registry Effort & ROI

Registry-based discovery was the most resource-intensive workstream with the lowest ROI. Of 743 registry records processed across FR and NL, only 4 yielded verified DJI Dock partners — all of which were already discovered by 3-4 other sources independently.

5.1 The Effort

Government registries (SIRENE in France, ILT/DCRO in Netherlands) required a multi-stage pipeline that is fundamentally different for each country:

ChallengeDetail
Source identificationEach country has different registries (SIRENE, KVK, Companies House, CASA). Some are open APIs, some password-protected, some require bulk downloads, some only offer paid access.
Data format inconsistencySIRENE uses NAF codes + SIREN IDs. NL ILT uses operator certificates + OA/STS categories. No standard schema across countries.
Waterfall logic differs per countryFR used a two-track waterfall (name-based Track A + NAF signal-based Track B). NL used a single registry dump with multi-source aggregation. Each country requires custom filter design.
API access & rate limitsSIRENE bulk API (INSEE), Apollo for enrichment, Serper for QA verification. Each has rate limits, auth requirements, and credit costs.
Low confidence funnelFR SIRENE: 29.3M entities → 16.8M active → ~1,400 name-matched → 378 scored → 188 enriched → 3 verified. Conversion: 0.02% from active pool.
Expensive enrichmentApollo API for website/LinkedIn discovery + Serper for DJI Dock QA. ~800 Serper credits consumed for NL 411-record batch alone.

5.2 TOFU → Waterfall → BOFU Pipeline

Top of Funnel (TOFU): Registry Filtering

Track A — Name-based: Filter SIRENE for "drone/uav/rpas/telepilot" in company name. FR yielded 1,391 → 1,122 after NAF blacklist. Catches: INSTADRONE, AZUR DRONES. Misses: Groupe Protect, Altametris (names don't contain drone keywords).

Track B — Signal-based: Filter by core NAF codes (71.12B engineering, 80.10Z security, 30.30Z aircraft) + external signal validation. Pool: 31,859 companies. Catches companies missed by name filter but requires cross-referencing with Google/DJI/LinkedIn signals.

Waterfall Scoring

Each record scored on: NAF code relevance (A/B/C tier) + keyword presence + employee band + legal form + company category. FR scoring produced 378 candidates from Track A (Wave 1). Only 6% of enriched records passed Serper DJI Dock verification.

5.3 Recent False Positive Case Study

When we ran the 70 highest-scored FR SIRENE companies (NAF 71.12B, engineering firms) through the DJI Dock Hunter utility, 2 were flagged as "verified":

CompanyWhat happenedRoot cause
NETMETIC Serper enrichment found journaldugeek.com as the company's website. QA searched site:journaldugeek.com "DJI Dock" and found a news article → false positive. Tiny company with no website. Google returned a tech news site instead. QA treated the news domain as the company's own site.
RETAILLEAU RENOVATIONS 44 Serper enrichment found entreprises.lefigaro.fr (Le Figaro's business directory). QA searched the directory domain → false positive. Company only exists in a directory listing. No real website. Directory domain passed QA because it has DJI-related content elsewhere.

Both records were deleted after investigation. These domains (journaldugeek.com, entreprises.lefigaro.fr) have been added to the mega-domain blocklist to prevent recurrence. This illustrates the core challenge with registry data: most SIRENE companies are micro-enterprises with no web presence, making enrichment unreliable and QA prone to false positives.

5.4 Registry Verdict

Recommendation: Use registries for enrichment, not discovery.
• Cross-reference verified Dock partners with registry for SIREN ID, NAF code, employee data, founding date
• Do NOT run full registries through the Hunter pipeline — <1% conversion, high false positive risk, expensive
• Each country's registry requires custom pipeline engineering (different APIs, schemas, access methods, filter logic)
• Intelligence sources (Comet, Google Search, AI research, DJI dealer list) discover the same companies at 100x lower cost

6. Key Insights

A. Intelligence sources have 100% conversion

Comet (32/32), Google Search (34/34), ChatGPT (33/33), Claude (11/11) — every company surfaced by these sources was DJI Dock verified. These sources pre-filter for relevance. This is the highest-ROI discovery channel.

B. Multi-source corroboration = strongest signal

27 companies appeared in 2+ independent sources. 100% are verified. When Comet, Google, AND ChatGPT independently flag a company → it's a Dock partner. Top examples: Escadrone (5 sources), Flying eye (5), Instadrone (5), Prodrones (5).

C. Google Search: best solo discovery, but needs cleanup

21 verified records were found ONLY by Google Search. However, Google Search results require significant post-processing:

  • International resellers appear in country-specific searches (~30-40% of results are from other countries)
  • Media/news entities must be filtered out (news articles mention DJI Dock but aren't companies)
  • Directory/aggregator sites can produce false positives (e.g., drones.nl directory listing treated as company website)
  • Domain validation is essential before running DJI Dock QA verification

All these filters are now built into the Dock Radar tooling (mega-domain blocklist, directory detection, shared-domain detection, media entity filter).

D. DJI Dealer List: high value, 100% conversion

100% conversion for both FR (12/12) and NL (7/7). All DJI Enterprise Dealers are now marked as verified — they are listed on the official DJI dealer page and authorized to sell all DJI Dock models. Note: 6 of 19 dealers were verified based on their DJI listing alone (no "DJI Dock" mention found on their own websites via programmatic QA). These are B2B distributors (security, surveying, topography) that sell DJI products as part of a broader catalog.
