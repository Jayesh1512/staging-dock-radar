3. Data Cleanup Applied

  • 1
    Cross-source domain merge — 11 companies appear in both Comet and DJI Reseller List with matching domains (e.g., Escadrone → escadrone.com, Flying Eye → flyingeye.fr, Abot → abot.fr). Merged into single grouped company, keeping best score from each source.
  • 2
    SIRENE registry overlap — Escadrone and Prodrones appear in SIRENE (govt_registry) with matching domains in Comet + DJI. These become 3-source companies after grouping. Domain-based dedup resolves different name formatting (e.g., "ESCADRONE" vs "Escadrone").
  • 3
    Google Search company names stored as page titles — Google Search crawler stores page titles instead of extracted company names. Fix pending: re-import with proper name extraction. Currently 0 Google Search records for France (import not yet run).
  • 4
    365 SIRENE records with no website/LinkedIn — The SIRENE registry provides business registration data only (SIREN ID, NAF code, employee band). No contact fields. These are not eliminated — they remain eligible for all tiers. Queued for enrichment via Apollo / LinkedIn search.
  • 5
    Entity type unknown for 90% of records — SIRENE does not classify companies as operator/reseller/SI. Only Comet (18 records) and DJI Reseller List (12 records) have entity_type populated. Classification requires enrichment or manual review.
  • 6
    Google Search normalized_name fix pending — When Google Search data is imported, the normalized_name extraction needs to strip page titles and extract the actual company name. This is a known fix queued for the re-import pipeline.
Enrichment approach: No company is excluded from ranking due to missing website or LinkedIn. Blank fields are enrichment opportunities, not elimination criteria. After the top 30 is finalized, we run enrichment (Apollo / LinkedIn lookup / manual search) to fill gaps before BD outreach.
